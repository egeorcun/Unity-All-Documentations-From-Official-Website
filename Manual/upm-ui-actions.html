    Adding and removing

    You can perform a variety of tasks through the Package Manager window:

    • Install a feature set from the Unity registry
    • Install a package from a registry
    • Install a package from a local folder
    • Install a package from a local tarball file
    • Install a package from a Git URL
    • Install a package from a registry by name
    • Remove an installed package from a project
    • Disable a built-in package
    • Switch to another package version
    • Import an Asset Store package into your project
    • Update an Asset Store package
    • Remove imported assets from a project
    • Delete a package from the Asset Store cache
    • Perform an action on multiple packages or feature sets

    The procedures described in these sections obscure a lot of the details of what the Package Manager is actually doing behind the scenes. The Package Manager window provides an intuitive interface to install and uninstall packages, by adding and removing packages as project dependencies in your project’s manifest, and then selecting the correct version to install. The version that Package Manager selects doesn’t always match the version you indicated. For more information, see Dependency and resolution.
